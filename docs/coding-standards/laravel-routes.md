# Laravel Routes

## Rules

### Controllers

Controllers are not part of the standard architecture
- Use Livewire components instead
- Exception: only when Livewire clearly does not make sense

### CRUD Architecture

Use **Laravel Front** for simple admin CRUDs. Use **Livewire** for create/edit only when:
- The form manages multiple related models inline
- It has dynamic add/remove rows
- It requires real-time field interactions

When Livewire overrides a Front route:
```php
Route::front('Team\\Product');
Route::livewire('/products/create', Livewire\Teams\ProductEdition::class)->name('products.create');
```

## Documentation

### About Laravel Front routes naming

Laravel Front registers named routes with the pattern `{prefix}.front.{resource-slug}`.
The index route has no suffix — the slug alone is the full name.

| Action  | Named route                        |
|---------|------------------------------------|
| index   | `admin.front.billing-plans`        |
| create  | `admin.front.billing-plans.create` |
| edit    | `admin.front.billing-plans.edit`   |

### Route Definitions

- Route files must minimize verbosity by defining a base namespace and using folder-based notation when possible.
- Route declarations must not repeatedly declare full class names when a namespace or folder-based route helper can express the same intent.
- Code must never invent route names. When a route name is referenced and there is uncertainty, verify it with `php artisan route:list` before using it.
- Code must use `route()` for URL references. Do not hardcode internal application paths as strings, including breadcrumb `url` values.

### Route File Separation

- Browser-facing routes must live in `routes/web.php` or the appropriate browser route file.
- API endpoints, webhooks, external service callbacks, and headless endpoints must live in `routes/api.php` or another API-specific route file.
- API or webhook routes must not be placed in `web.php`; they do not need session/cookie middleware and should not require CSRF exceptions because they were placed in the wrong file.
- When reviewing route files, flag any route whose middleware/file placement does not match how the endpoint is consumed.

### Laravel Front Route Names

- When code references a route generated by Laravel Front, it must use Laravel Front's route name format: `{prefix}.front.{resource-slug}` for the index route, `{prefix}.front.{resource-slug}.create` for create, and `{prefix}.front.{resource-slug}.edit` for edit.
- Code must not append `.index` to Laravel Front index routes.
- Code must not use bare resourceful route names like `admin.billing-plans.index` for Laravel Front resources.
- Code must not invent a named `show` route for Laravel Front resources; use the configured `base_url` directly when a show page is needed.
